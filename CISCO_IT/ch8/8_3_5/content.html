8.3 Descrever conceitos e tecnologias de rede básica  
  8.3.5 Definir ICMP  

O Internet Control Message Protocol (ICMP, protocolo de mensagens de controle da Internet) é usado por dispositivos em uma rede para enviar mensagens de controle e de erro a computadores e servidores. Há diversos usos para o ICMP, como anunciar erros e congestionamentos de rede e apresentar soluções de problemas.

O recurso do packet internet groper (ping, procurador de pacotes na Internet) é comumente usado para testar conexões entre computadores. O ping é um utilitário simples, embora altamente útil, de linha de comando usado para determinar se um endereço IP específico está acessível. O ping pode ser usado com o endereço IP para testar a conectividade do IP. O ping funciona enviando uma solicitação de eco do ICMP para um computador de destino ou outro dispositivo de rede. Então, o dispositivo receptor retorna uma mensagem de resposta do eco do ICMP para confirmar a conectividade.

O ping é uma ferramenta de solução de problemas usada para determinar a conectividade básica. As trocas de linha de comando que podem ser usadas com o comando ping são mostradas na Figura 1. As solicitações de eco do ICMP (pings) são enviadas para o computador de destino. Se for atingível, o computador de destino responde com quatro respostas de eco do ICMP. O percentual de respostas bem-sucedidas pode ajudá-lo a determinar a confiabilidade e a acessibilidade do computador de destino.

Você também pode usar o ping para localizar o endereço IP de um host quando o nome é conhecido. Se você utilizar o ping para localizar o nome de um site, por exemplo, www.cisco.com, conforme mostrado na Figura 2, o endereço IP do servidor será exigido.

Outras mensagens do ICMP são usadas para reportar pacotes não entregues, dados em uma rede IP que incluam endereços IP de origem e de destino e se um dispositivo está ocupado demais para manipular o pacote. Os dados, na forma de um pacote, chegam em um roteador, que é um dispositivo de rede que encaminha pacotes de dados pelas redes a seus destinos. Se o roteador não souber para onde enviar o pacote, o roteador o excluirá. Então, o roteador retornará uma mensagem do ICMP para o computador remetente informando que os dados foram excluídos. Quando ficar muito ocupado, um roteador poderá enviar uma mensagem do ICMP diferente para o computador remetente indicando que esse computador deverá diminuir a velocidade pois há um congestionamento na rede.

8.5 Descrever as topologias e arquiteturas da LAN  
  8.5.2 Descrever as arquiteturas da LAN  

A arquitetura da LAN descreve as topologias física e lógica usadas em uma rede. A Figura 1 mostra as três arquiteturas de LAN mais comuns.

Ethernet
A arquitetura Ethernet é baseada no padrão IEEE 802.3. O padrão IEEE 802.3 especifica que uma rede use o método de controle de acesso CSMA/CD (Carrier Sense Multiple Access with Collision, acesso múltiplo de detecção de portadora com detecção de colisão). No CSMA/CD, os hosts acessam a rede usando o método de topologia de transmissão "primeiro a chegar, primeiro a ser servido" para transmitir dados.

A Ethernet usa uma topologia lógica de transmissão ou de barramento e uma topologia física de barramento ou estrela. Conforme as redes se expandem, a maioria das redes de Ethernet é implementada usando uma topologia estrela hierárquica ou estrela estendida, conforme mostrado na Figura 1. As taxas de transferência padrão são de 10 Mbps e 100 Mbps, mas os novos padrões esboçam o Gigabit Ethernet, que é capaz de atingir velocidades de até 1.000 Mbps (1 Gbps).

Token Ring
Originalmente, a IBM desenvolveu o Token Ring como uma arquitetura de rede confiável baseada no método de controle de acesso de passagem de token. O Token Ring é, geralmente, integrado aos sistemas de mainframe da IBM. O Token Ring é usado com computadores e mainframes.

O Token Ring é um exemplo de uma arquitetura na qual a topologia física é diferente de sua topologia lógica. A topologia Token Ring é referida como um anel conectado em estrela, pois a aparência externa do projeto da rede é de uma estrela. Os computadores se conectam a um hub central, chamado de unidade de acesso multiestação (MSAU, multistation access unit). Dentro do dispositivo, no entanto, a fiação forma um caminho de dados circular, criando um anel lógico. O anel lógico é criado pelo token viajando de uma porta MSAU para um computador. Se o computador não tiver qualquer dado para enviar, o token é devolvido para a porta MSAU e, depois, sai para a próxima porta para o computador seguinte. Esse processo continua em todos os computadores e, dessa forma, assemelha-se a um anel físico.

FDDI
O FDDI é um tipo de rede de Token Ring. A implementação e a topologia do FDDI diferem da arquitetura de LAN Token Ring da IBM. O FDDI é, geralmente, usado para conectar várias construções em um complexo de escritórios ou em um campus universitário.

O FDDI é executado em fibra óptica. O FDDI combina desempenho de alta velocidade com as vantagens da topologia em anel de passagem de token. O FDDI é executado a 100 Mbps em uma topologia de anel duplo. O anel externo é chamado de anel primário, e o anel interno é chamado de anel secundário.

Normalmente, o tráfego flui somente no anel primário. Se o anel primário falhar, os dados automaticamente fluirão no anel secundário na direção oposta.

Uma rede FDDI de anel duplo suporta um máximo de 500 computadores por anel. A distância total de cada extensão do anel do cabo é de 100 km (62 milhas). Um repetidor, que é um dispositivo que gera novamente os sinais, é necessário a cada 2 km (1,2 milhas). Recentemente, muitas redes de token ring foram substituídas por redes Ethernet mais rápidas.

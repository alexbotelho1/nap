8.3 Descrever conceitos e tecnologias de rede básica  
  8.3.2 Descrever o endereçamento IP  

Um endereço IP é um número usado para identificar um dispositivo na rede. Cada dispositivo em uma rede deve ter um único endereço IP para se comunicar com outros dispositivos de rede. Conforme observado anteriormente, um host é um dispositivo que envia ou recebe informações na rede. Os dispositivos de rede são dispositivos que movem os dados pela rede, incluindo hubs, comutadores e roteadores. Em uma LAN, cada host e dispositivo de rede deve ter um endereço IP dentro da mesma rede para poderem se comunicar entre si.

O nome de uma pessoa e suas impressões digitais normalmente não mudam. Eles fornecem um rótulo ou endereço para o aspecto físico da pessoa – o corpo. O endereço de correspondência de uma pessoa, por outro lado, é relativo ao lugar onde a pessoa vive ou recolhe suas correspondências. Esse endereço pode mudar. Em um host, o endereço MAC (Media Access Control, controle de acesso à mídia) (explicado abaixo) é atribuído à NIC do host e é conhecido como o endereço físico. O endereço físico permanece o mesmo, independentemente de onde o host é colocado na rede, da mesma maneira que as impressões digitais permanecem com a pessoa, independentemente de onde ela vá.

O endereço IP é semelhante ao endereço de correspondência de uma pessoa. Ele é conhecido como um endereço lógico porque é atribuído de maneira lógica com base no local do host. O endereço IP, ou endereço de rede, é baseado na rede local e é atribuído a cada host por um administrador da rede. Esse processo é semelhante ao governo local que atribui um endereço de rua com base na descrição lógica da cidade ou povoado e da vizinhança.

Um endereço IP consiste de uma série de 32 bits binários (uns e zeros). É muito difícil para as pessoas lerem um endereço IP binário. Por essa razão, os 32 bits são agrupados em quatro bytes de 8 bits chamados octetos. Um endereço IP, mesmo nesse formato agrupado, é difícil para as pessoas lerem, escreverem e lembrarem; portanto, cada octeto é apresentado como seu valor decimal, separado por um ponto decimal ou ponto. Esse formato é conhecido como uma notação decimal com ponto. Quando um host é configurado com um endereço IP, ele é inserido como um número decimal com ponto, como 192.168.1.5. Imagine se você tivesse que inserir o binário de 32 bits equivalente a isto: 11000000101010000000000100000101. Se apenas um bit fosse digitado incorretamente, o endereço seria diferente e o host não poderia se comunicar na rede.

O endereço IP lógico de 32 bits é hierárquico e composto de duas partes. A primeira parte identifica a rede e a segunda parte identifica um host naquela rede. As duas partes são exigidas em um endereço IP. Como exemplo, se um host possui o endereço IP 192.168.18.57, os três primeiros octetos, 192.168.18, identificam a porção da rede do endereço, e o último octeto, 57, identifica o host. Isso é conhecido como endereçamento hierárquico, porque a porção da rede indica a rede na qual cada endereço de host único está localizado. Os roteadores precisam apenas saber como chegar em cada rede e não no local de cada host individual.

Os endereços IP são divididos nas cinco classes a seguir:

  • Classe A – Redes grandes, implementadas por grandes empresas e alguns países
  • Classe B – Redes de tamanho médio, implementadas por universidades
  • Classe C – Redes pequenas, implementadas por provedores de serviços de Internet para assinaturas de clientes
  • Classe D – Uso especial para multidifusão
  • Classe E – Usada para testes experimentais

Máscara de sub-rede
Uma máscara de sub-rede é usada para indicar a porção da rede de um endereço IP. Como o endereço IP, a máscara de sub-rede é um número decimal com ponto. Normalmente todos os hosts dentro de uma LAN usam a mesma máscara de sub-rede. A Figura 1 mostra as máscaras de sub-rede padrão para endereços IP úteis mapeados para as três primeiras classes de endereços IP:

  • 255.0.0.0 – Classe A, que indica que o primeiro octeto do endereço IP é a porção da rede
  • 255.255.0.0 – Classe B, que indica que os dois primeiros octetos do endereço IP são a porção da rede
  • 255.255.255.0 – Classe C, que indica que os três primeiros octetos do endereço IP são a porção da rede

Se uma organização possui uma rede de Classe B, mas precisa fornecer endereços IP para quatro LANs, a organização teria que subdividis o endereço da Classe B em quatro partes menores. A criação de sub-redes é uma divisão lógica de uma rede. Ela fornece os meios para dividir uma rede e a máscara de sub-rede especifica como ela é subdividida. Um administrador de rede experiente normalmente executa a criação de sub-redes. Após o esquema de criação de sub-redes ter sido criado, os endereços IP adequados e as máscaras de sub-rede podem ser configuradas nos hosts nas quatro LANs. Essas habilidades são ensinadas nos cursos do Cisco Networking Academy relacionados às habilidades de rede de nível CCNA.

Configuração manual
Em uma rede com um pequeno número de hosts, é fácil configurar manualmente cada dispositivo com o endereço IP adequado. Um administrador de rede que entende de endereçamento de IP deve atribuir os endereços e deve saber como escolher um endereço válido para uma determinada rede. O endereço IP inserido é único para cada host dentro da mesma rede ou sub-rede.

Para inserir manualmente um endereço IP em um host, vá até as configurações de TCP/IP na janela de Propriedades da NIC (Network Interface Card, placa de interface de rede). A NIC é o hardware que permite a um computador se conectar a uma rede. Ela possui um endereço chamado endereço MAC. Enquanto o endereço IP é um endereço lógico definido pelo administrador da rede, um endereço MAC é "gravado" ou permanentemente programado na NIC quando ela é fabricada. O endereço IP de uma NIC pode ser alterado, mas o endereço MAC nunca é alterado.

A diferença principal entre um endereço IP e um endereço MAC é que o MAC é usado para entregar quadros na LAN, enquanto o IP é usado para transportar quadros fora da LAN. Um quadro é um pacote de dados, juntamente com informações do endereço adicionadas ao início e final do pacote antes da transmissão pela rede. Depois que um quadro é entregue à LAN de destino, o endereço MAC é usado para entregar o quadro ao host final naquela LAN.

Se mais de poucos computadores compuserem uma LAN, configurar manualmente endereços IP para cada host da rede pode levar muito tempo e pode ser uma atividade propensa a erros. Neste caso, o uso de um servidor DHCP (Dynamic Host Configuration Protocol, protocolo de configuração dinâmica de hosts) atribuiria automaticamente endereços IP e simplificaria bastante o processo de endereçamento.

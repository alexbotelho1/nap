8.9 Descrever como configurar uma NIC e um modem  
  8.9.3 Descrever a instalação de um modem  

Um modem é um dispositivo eletrônico que transfere dados entre computadores usando sinais análogos em uma linha telefônica. Exemplos de modems são mostrados na Figura 1. O modem converte os dados digitais em sinais análogos para transmissão. O modem na extremidade receptora converte novamente os sinais análogos em dados digitais a ser interpretados pelo computador. O processo de conversão de sinais análogos em digitais e, depois, de digitais em análogos é chamado de modulação/demodulação. A transmissão baseada em modem é bastante precisa, apesar do fato de que as linhas telefônicas podem apresentar ruídos devido a cliques, estática e outros problemas.

Um modem interno é conectado a um slot de expansão na placa-mãe. Para configurar um modem, deverão ter sido configurados os jumpers para selecionar endereços IRQ e de E/S. Não é necessário nenhuma configuração para um modem plug-and-play, que só pode ser instalado em uma placa-mãe que suporte plug-and-play. Deve ser configurado um modem que use uma porta serial que ainda não esteja sendo utilizada. Além disso, os drivers de software que vêm com o modem devem ser instalados para o modem funcionar adequadamente. Os drivers de modems são instalados do mesmo modo como são instalados os drivers para NICs.

Modems externos se conectam a um computador por meio das portas serial e USB.

Quando os computadores usam o sistema de telefonia pública para se comunicar, temos uma rede discada (DUN, dial-up networking). Os modems se comunicam uns com os outros usando sinais de tom de áudio. Isso significa que os modems são capazes de duplicar as características de discagem de um telefone. A DUN cria uma conexão Point-to-Point Protocol (PPP, protocolo ponto a ponto) entre dois computadores em uma linha telefônica.

Depois de ter sido estabelecida a conexão, ocorre uma "seqüência de comunicação" entre os dois modems e os computadores. A seqüência de comunicação é uma série de comunicações breves que ocorrem entre os dois sistemas. Isso é feito para estabelecer a prontidão dos dois modems e dos computadores para iniciar a troca de dados. Modems de discagem enviam dados em uma linha telefônica serial na forma de um sinal análogo. Uma vez que os sinais análogos se alteram gradual e continuamente, os sinais podem ser reproduzidos como ondas. Nesse sistema, os sinais digitais são representados por números 1 e 0. Os sinais digitais devem ser convertidos em uma forma de onda para viajar em linhas telefônicas. Os números 1 e 0 são revertidos à forma digital pelo modem receptor para que o computador receptor possa processar os dados.

Comandos AT
Todos os modems requerem software para controlar a sessão de comunicação. A maioria dos softwares de modem usa o conjunto de comandos compatíveis com Hayes. O conjunto de comandos Hayes é baseado em um grupo de instruções que sempre começam com um conjunto de caracteres de atenção (AT), seguido pelos caracteres de comando. Esses são conhecidos como comandos AT. O comando AT é mostrado na Figura 2.

Os comandos AT são comandos de controle do modem. O conjunto de comandos AT é usado para emitir ao modem as instruções de discagem, desligamento, redefinição, entre outras. A maioria dos manuais dos usuários que vêm com um modem contém uma lista completa do conjunto de comandos AT.

O código-padrão compatível com Hayes a ser discado é ATDxxxxxxx. Geralmente, não há espaços em uma string AT. Se for inserido um espaço, a maioria dos modems o ignorará. O "x" significa o número discado. Haverá sete dígitos para uma ligação local e 11 dígitos para uma ligação de longa distância. Um W indica que o modem aguardará por uma linha externa, se necessário, para estabelecer um tom antes de prosseguir. Às vezes, é adicionado um T para significar uma discagem de tom, ou um P para significar uma discagem de pulso.

8.8 Explicar os modelos de dados OSI e TCP/IP  
  8.8.2 Definir o modelo OSI  

O modelo OSI é uma estrutura padrão da indústria usada para dividir as comunicações de rede em sete camadas distintas. Embora existam outros modelos, a maioria dos fornecedores de rede atuais criam seus produtos usando essa estrutura.

Um sistema que implementa um comportamento de protocolos consistindo em uma série dessas camadas é conhecido como uma pilha de protocolos. As pilhas de protocolos podem ser implementadas no hardware, no software, ou em uma combinação de ambos. Geralmente, só as camadas inferiores são implementadas no hardware, e as camadas superiores são implementadas no software.

Cada camada é responsável por parte do processo de preparação dos dados para transmissão na rede. O gráfico na Figura 1 mostra o que cada camada do modelo OSI faz.

No modelo OSI, quando os dados são transferidos, diz-se que os dados viajam virtualmente para baixo nas camadas do modelo OSI do computador remetente, e para cima nas camadas do modelo OSI do computador receptor.

Quando um usuário quer enviar dados, como um e-mail, o processo de encapsulamento começa na camada Aplicativo. A camada Aplicativo é responsável por fornecer o acesso dos aplicativos à rede. As informações fluem pelas três camadas superiores e são consideradas dados quando chegam embaixo na camada Transporte.

Na camada Transporte, os dados são divididos em segmentos mais gerenciáveis, ou unidades de dados de protocolo (PDUs, protocol data units) da camada Transporte, para um transporte ordenado pela rede. Uma PDU descreve os dados conforme se move de uma camada do modelo OSI para outra. A PDU da camada Transporte também contém informações, como números de portas, números de seqüências e números de reconhecimentos, que são usadas para um transporte confiável de dados.

Na camada Rede, cada segmento da camada Transporte se torna um pacote. O pacote contém endereçamento lógico, além de outras informações de controle da camada 3.

Na camada Link de Dados, cada pacote da camada Rede se torna um quadro. O quadro contém informações sobre o endereço físico e sobre correção de erros.

Na camada Física, o quadro é transformado em bits. Esses bits são transmitidos um por vez através de uma mídia de rede.

No computador receptor, o processo de desencapsulamento reverte o processo de encapsulamento. Os bits chegam na camada Física do modelo OSI do computador receptor. O processo de viajar virtualmente para cima no modelo OSI do computador receptor trará os dados para a camada Aplicativo, onde um programa de e-mail exibirá o e-mail.

OBSERVAÇÃO: Mnemônicos podem ajudá-lo a lembrar as sete camadas do OSI. Alguns exemplos incluem: "Falei lentamente rindo tudo sobre andorinhas amarelas" e "Águas azuis sobre terrenos retos lentamente fluem".

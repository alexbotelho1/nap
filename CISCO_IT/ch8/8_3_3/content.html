8.3 Descrever conceitos e tecnologias de rede básica  
  8.3.3 Definir DHCP  

DHCP (Dynamic Host Configuration Protocol, protocolo de configuração dinâmica de hosts) é um utilitário de software usado para atribuir dinamicamente endereços IP a dispositivos de rede. Esse processo dinâmico elimina a necessidade de atribuir manualmente endereços IP. Um servidor DHCP pode ser configurado e os hosts podem ser configurados para obterem automaticamente um endereço IP. Quando um computador é configurado para obter um endereço IP automaticamente, todas as outras caixas de configuração de endereçamento IP são escurecidas, conforme exibido na Figura 1. O servidor mantém uma lista de endereços IP para atribuir e gerencia o processo para que cada dispositivo da rede receba um único endereço IP. Cada endereço é mantido por um período de tempo predeterminado. Quando o tempo expira, o servidor DHCP pode usar esse endereço para qualquer computador que ingressar na rede.

Estas são as informações do endereço IP que um servidor DHCP pode atribuir aos hosts:

  • Endereço IP
  • Máscara de sub-rede
  • Gateway padrão
  • Valores opcionais, como endereço do servidor DNS (Domain Name System, sistema de nomes de domínio)

O servidor DHCP recebe uma solicitação de um host. O servidor seleciona as informações do endereço IP a partir de um conjunto de endereços predefinidos armazenados em um banco de dados. Depois que as informações do endereço IP forem selecionadas, o servidor DHCP irá oferecer esses valores ao host solicitante da rede. Se o host aceitar a oferta, o servidor DHCP aluga o endereço IP por um período de tempo específico.

Usar um servidor DHCP simplifica a administração de uma rede porque o software controla os endereços IP. Configurar automaticamente TCP/IP também reduz a possibilidade de atribuir endereços IP duplicados ou inválidos. Antes que um computador da rede possa aproveitar os serviços do servidor DHCP, o computador deve poder identificar o servidor na rede local. Um computador pode ser configurado para aceitar um endereço IP de um servidor DHCP clicando na opção "Obter um endereço IP automaticamente" na janela de configuração da NIC, conforme exibido na Figura 2.

Se seu computador não puder se comunicar com o servidor DHCP para obter um endereço IP, o sistema operacional Windows irá atribuir automaticamente um endereço IP privado. Se seu computador receber um endereço IP no intervalo de 169.254.0.0 a 169.254.255.255, o computador poderá apenas se comunicar com outros computadores do mesmo intervalo. Um exemplo de quando esses endereços privados seriam úteis é no laboratório de uma sala de aula onde você deseja impedir o acesso fora da rede. Esse recurso do sistema operacional é chamado de APIPA (Automatic Private IP Addressing, endereçamento IP privado automático). O APIPA solicitará continuamente um endereço IP a partir de um servidor DHCP para o seu computador.

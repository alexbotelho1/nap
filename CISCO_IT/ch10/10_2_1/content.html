10.2 Descrever boas habilidades de comunicação e de comportamento profissional  
  10.2.1 Determinar o problema do computador do cliente  

Uma das primeiras tarefas do técnico é determinar o tipo de problema do computador que o cliente está tendo.

Lembre-se destas três regras do início de sua conversa:

  • Conhecer - Chame seu cliente pelo nome
  • Relacionar - Use comunicação sucinta para criar uma ligação exclusiva entre você e seu cliente
  • Entender - Determine o nível de conhecimento do cliente sobre o computador, para saber como se comunicar de maneira eficaz com o cliente

Para fazer isso, você deve praticar habilidades de escuta ativa. Deixe que o cliente conte toda a estória. Durante o período em que o cliente estiver explicando o problema, de vez em quando fale algumas pequenas palavras ou frases, como "Entendo", "Sim", "Compreendo" ou "Ok". Esse comportamento permite que o cliente saiba que você está lá e que você está escutando. É diferente de interromper o cliente para fazer uma pergunta ou uma afirmação.

Um técnico não deve interromper o cliente para fazer uma pergunta ou uma afirmação. É indelicado, desrespeitoso e cria tensão. Muitas vezes em uma conversa, você se depara pensando no que vai dizer, antes de a outra pessoa terminar de falar. Quando você faz isso, você realmente não está escutando. Como prática, tente escutar atenciosamente quando as outras pessoas falam e deixe-as terminarem suas considerações.

Depois que escutar o cliente explicar todo o problema, explique o que o cliente disse. Isso ajudará a convencer o cliente de que você escutou e entendeu a situação. Uma boa prática para esclarecer é parafrasear a explicação do cliente iniciando com as palavras "Deixe me ver se eu entendi o que você disse..." Essa é uma ferramenta muito eficaz que mostra ao cliente que você escutou e está preocupado com os problemas.

Depois que você garantir ao cliente que entendeu o problema, você provavelmente terá que fazer algumas perguntas de acompanhamento. Certifique-se de que essas perguntas são apropriadas. Não faça perguntas que o cliente já tenha respondido ao descrever o problema. Isso somente irritará o cliente e mostrará que você não estava escutando.

As perguntas de acompanhamento devem ser perguntas finais, direcionadas, com base nas informações já obtidas. As perguntas finais devem ser concentradas na obtenção de informações específicas. O cliente deve ser capaz de responder com um simples "sim" ou "não" ou com uma resposta real, como, "Windows XP Pro". Use todas as informações obtidas do cliente para continuar preenchendo a ordem de trabalho.

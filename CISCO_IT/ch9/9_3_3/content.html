9.3 Identificar procedimentos de segurança  
  9.3.3 Descrever formas de proteger os dados  

O valor dos equipamentos físicos, geralmente, é muito menor do que o valor dos dados que os equipamentos contêm. A perda de dados sensíveis para um concorrente da empresa ou para criminosos pode ser dispendiosa. Essas perdas podem resultar em uma falta de confiança na empresa e na demissão dos técnicos de informática responsáveis pela segurança dos computadores. Para proteger os dados, há vários métodos de proteção da segurança que podem ser implementados.

Proteção de senhas
A proteção de senha pode evitar acesso não-autorizado ao conteúdo, conforme mostrado na Figura 1. Os invasores são capazes de obter acesso a dados não-protegidos do computador. Todos os computadores deverão ser protegidos por senha. Dois níveis de proteção de senha são recomendados:

  • BIOS – Impede que as definições do BIOS sejam alteradas sem a senha apropriada
  • Login – Impede o acesso não-autorizado à rede

Os logins de rede fornecem um meio de registrar a atividade na rede e, também, de impedir ou permitir acesso aos recursos. Isso possibilita determinar que recursos estão sendo acessados. Geralmente, o administrador do sistema define uma convenção de nomeação para os nomes de usuários ao criar logins de rede. Um exemplo comum de um nome de usuário é a primeira inicial do nome da pessoa seguida do sobrenome completo. Você deverá simplificar a convenção de nomeação de nome de usuário para que as pessoas não tenham dificuldade para lembrá-la.

Ao atribuir senhas, o nível de controle de senha deverá corresponder ao nível de proteção exigido. Uma boa política de segurança deverá ser estritamente reforçada e incluir, sem limitação, as seguintes regras:

  • As senhas deverão expirar após um período específico de tempo.
  • As senhas deverão conter uma combinação de letras e números a fim de que não possam ser facilmente descobertas.
  • Os padrões de senhas deverão evitar que os usuários escrevam as senhas e as deixem desprotegidas do público.
  • As regras sobre expiração e bloqueio de senhas deverão ser definidas. As regras de bloqueio se aplicam quando foi feita uma tentativa frustrada de acessar o sistema ou quando foi detectada uma alteração específica na configuração do sistema.

Para simplificar o processo de administração da segurança, é comum atribuir usuários a grupos e, depois, atribuir os grupos a determinados recursos. Isso permite que a capacidade de acesso dos usuários em uma rede seja facilmente alterada atribuindo ou removendo o usuário dos vários grupos. Isso é útil ao configurar contas temporárias para colaboradores ou consultores visitantes, dando a você a capacidade de limitar o acesso aos recursos.

Criptografia de dados
A criptografia de dados usa códigos e cifras. O tráfego entre recursos e computadores na rede pode ser protegido dos invasores monitorando ou registrando as transações por meio da implementação de criptografia. Não será possível decifrar os dados capturados a tempo de se fazer uso deles.

Uma Virtual Private Network (VPN, rede particular virtual) usa a criptografia para proteger dados. Uma conexão VPN permite que um usuário remoto acesse com segurança os recursos como se seu computador estivesse fisicamente conectado à rede local.

Proteção de portas
Cada comunicação que use TCP/IP é associada a um número de porta. HTTPS, por exemplo, usa a porta 443 por padrão. Um firewall, conforme mostrado na Figura 2, é um modo de proteger um computador de invasão através das portas. O usuário pode controlar os tipos de dados enviados a um computador selecionando que portas serão abertas e que portas serão protegidas. Os dados que são transportados em uma rede são chamados de tráfego.

Backup de dados
Os procedimentos de backup de dados deverão ser incluídos em um plano de segurança. Os dados podem ser perdidos ou danificados em circunstâncias como roubo ou falha do equipamento, ou em um acidente, como incêndio ou inundação. Fazer backup dos dados é um dos modos mais eficazes de proteger os dados contra perda. A seguir, são dadas algumas considerações sobre os backups de dados:

  • Freqüência dos backups – Os backups podem levar um longo tempo para serem executados. Às vezes, é mais fácil fazer um backup completo mensal ou semanal e, depois, fazer backups parciais freqüentes de qualquer dado que tenha sido alterado desde o último backup completo. No entanto, espalhar os backups em vários registros aumenta a quantidade de tempo necessária para recuperar os dados.
  • Armazenagem de backups – Os backups deverão ser transportados para um local de armazenagem fora do local para segurança extra. A mídia do backup atual deve ser transportada para fora do local do escritório em uma base diária, semanal ou mensal, conforme exigido pela organização local.
  • Segurança dos backups – Os backups podem ser protegidos por senhas. Essas senhas deverão ser informadas antes que os dados na mídia de backup possam ser recuperados.

Segurança do sistema de arquivos
Todos os sistemas de arquivos mantêm controle dos recursos, mas só os sistemas de arquivos com registro de diários podem registrar o acesso por usuário, data e hora. O sistema de arquivos FAT 32, mostrado na Figura 3, que é usado em algumas versões do Windows, não traz os recursos de criptografia e de registro de diário. Como resultado, as situações que requerem uma boa segurança são, geralmente, enfrentadas usando um sistema de arquivo como o NTFS, que faz parte do Windows 2000 e do Windows XP. Se for necessário uma segurança maior, será possível executar certos utilitários, como o CONVERT, para atualizar um sistema de arquivos FAT 32 para o NTFS. O processo de conversão não é reversível. É importante definir claramente seus objetivos antes de fazer a transação.

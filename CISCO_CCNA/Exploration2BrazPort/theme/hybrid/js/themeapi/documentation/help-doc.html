Como este documento de API é organizado

Este documento de API (Application Programming Interface, Interface de programação de aplicativos) tem páginas correspondentes aos itens na barra de navegação, descrito da seguinte forma.

Classe

Cada classe tem sua própria página separada. Cada uma dessas páginas tem três seções que consistem em uma descrição de classe, tabelas de resumo e descrições de membro detalhadas:

  • Diagrama de herança de classe
  • Subclasses diretas
  • Declaração de classe
  • Descrição de classe

  • Resumo do campo
  • Resumo do construtor
  • Resumo do método

  • Detalhes do campo
  • Detalhes do construtor
  • Detalhes do método
Cada entrada de resumo contém a primeira sentença da descrição detalhada do item.

Índice

O Índice contém uma lista alfabética de todas as classes, construtores, métodos e campos.

Anterior/Próximo

Estes links levam você para a próxima classe ou anterior, interface, pacote ou página relacionada.

Quadros/Sem quadros

Estes links mostram e ocultam os quadros HTML. Todas as páginas estão disponíveis com ou sem quadros.

Este arquivo de ajuda se aplica à documentação de API gerada usando o doclet padrão.
